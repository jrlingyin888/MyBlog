## 1. 目标

给博客加一个**轻音乐播放器**，为阅读营造放松氛围，同时绝不打扰不想听的人。

- 默认不出声；用一句温柔的提示邀请用户主动开启。
- 用户随时一键暂停（即「静音」）。
- 站内翻页时音乐不间断。
- 视觉与「慢一点」暖色杂志风一致，并带细腻动画。

**非目标（本版不做）**
- 不自动播放出声（浏览器规则 + 体验考量）。
- 不做歌单 / 多曲切换 / 播放进度条 / 音量滑块（保留为未来增量）。
- 不内置任何受版权音乐；音频由作者自备（免版税 / CC0）。

---

## 2. 关键约束

- **浏览器禁止「带声音的自动播放」**：必须用户交互后才能出声 → 默认关闭、点击开启。
- **多页站点整页刷新会打断音乐** → 用 Astro 视图过渡（`ClientRouter`）+ `transition:persist` 实现站内翻页续播。
- **版权**：音频文件由作者提供；文件缺失时优雅降级，不报错。

---

## 3. 行为规格

- **初始**：不播放。`<audio loop preload="none">`，音量默认 `0.35`（「轻」）。
- **位置**：右下角悬浮圆形按钮（锈橙底 `#b5542f` + 奶油音符 `#faf5ec`）。
- **首次提示**：按钮旁出现提示气泡「要不要来点轻音乐，放松一下」，带 `×` 关闭。
  - 关闭过气泡 **或** 点过播放 → 之后不再出现（`localStorage: mp-hint-dismissed`）。
- **点击按钮**：
  - 关 → 开：调用 `audio.play()`；成功则进入「播放中」（声波动画），`aria-label` 改为「暂停轻音乐」，记 `localStorage: mp-on=1`，隐藏气泡。
  - 开 → 关：`audio.pause()`；回到「待机」（音符 + 呼吸动画），记 `mp-on=0`。
- **持久化**：`localStorage` 记 `mp-on` 与 `mp-hint-dismissed`。
- **站内翻页**：视图过渡下播放器 `transition:persist` 保活，音乐不中断（见 §5）。
- **首屏 / 硬刷新 / 外链进入**：若 `mp-on=1`，加载后尝试 `audio.play()`；浏览器允许则续上，被拦截则保持待机、等用户再点一下（不报错、不打扰）。

---

## 4. 视觉与动画

- 圆按钮 58px，锈橙底，奶油 Tabler 音符图标；播放中切换为 3 根跳动声波条。
- **待机**：呼吸脉动（`scale 1 ↔ 1.07`，约 3s）。
- **提示气泡**：`#fffdf8` 底 + `#e6dcc8` 细边 + 指向按钮的小尾巴；淡入 + 上滑出现。
- **点击**：水波纹（ripple）扩散；音符 ↔ 声波切换。
- 配色为品牌暖色，**硬编码 hex，不随深色模式翻转**。
- **`prefers-reduced-motion: reduce`**：关闭呼吸 / 声波 / 水波纹动画，仅保留状态切换，保证无障碍与不晕动。

## 5. 架构 / 组件

- **新增 `src/components/MusicPlayer.astro`**（自包含）：
  - 整个组件标 `transition:persist="music-player"`，跨页不重建、音乐不断。
  - 结构：右下角定位容器 > 提示气泡 + 圆按钮（音符 + 声波）+ `<audio loop preload="none" src="/music/ambient.mp3">`。
  - 作用域 `<style>`：定位、品牌配色、动画 keyframes、reduced-motion 降级。
  - 内联 `<script>`：读写 `localStorage`、绑定点击、`play/pause`、捕获 `play()` 失败、首载尝试续播、提示显隐。脚本对视图过渡**幂等**（在 `astro:page-load` 重新绑定，状态以持久化的 `audio` 元素为准，避免重复播放或重复绑定）。
- **修改 `src/layouts/BaseLayout.astro`**：
  - `<head>` 引入并加入 `<ClientRouter />`（来自 `astro:transitions`）。
  - `<body>` 末尾加 `<MusicPlayer />`。
- **资源**：`public/music/ambient.mp3`（占位，作者后补）；`public/music/` 目录需存在（放 `.gitkeep` 占位）。

## 6. 占位音频处理（本版采用）

- 真曲子未就位时，`audio` 的 `src` 指向尚不存在的 `/music/ambient.mp3`。
- 点击播放 → `play()` 会失败（404 / 不可解码）→ 捕获后：不抛错，回到待机，显示一次性小提示「轻音乐即将上线」（约 2–3 秒后消失）。
- 作者把 `ambient.mp3` 放进 `public/music/` 后**自动生效，无需改代码**。

---

## 7. 无障碍

- 按钮 `aria-label` 随状态切换（「播放轻音乐」/「暂停轻音乐」）；键盘可聚焦，Enter / Space 触发。
- 气泡 `×` 有 `aria-label`；气泡不抢焦点。
- 尊重 `prefers-reduced-motion`。

---

## 8. 验收标准

- 右下角出现音乐按钮，待机有呼吸动画。
- 首次访问显示提示气泡；点 `×` 关闭后刷新不再出现；点播放后气泡消失且不再出现。
- 点击在「待机 / 播放」间切换，有水波纹 + 声波动画。
- 站内翻页（点导航 / 文章卡片）音乐不中断（放入真曲后验证）。
- 占位阶段点击不报错，显示「轻音乐即将上线」。
- `prefers-reduced-motion` 下动画降级。
- `mp-on` 与 `mp-hint-dismissed` 在 `localStorage` 持久化。
- `npm run build` 通过、`astro check` 0 错。
